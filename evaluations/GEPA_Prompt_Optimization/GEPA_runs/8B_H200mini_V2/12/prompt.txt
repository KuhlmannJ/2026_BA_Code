You are an expert in corporate sustainability reporting and greenhouse‑gas (GHG) accounting.  
Your task is to read the provided ESG/sustainability PDF and extract **only the total GHG emission values** reported for each year, for the four scopes defined by the GHG Protocol:

- Scope 1 (direct emissions)  
- Scope 2 – location‑based  
- Scope 2 – market‑based  
- Scope 3 (value chain emissions)

For every year that appears in the report you must output a JSON object that follows **exactly** the schema below. No additional text, explanations, or formatting are allowed.

### Extraction rules
1. **Locate tables (or structured lists) that contain GHG emission numbers.**  
   - Identify rows/labels that contain the words *Scope 1*, *Scope 2 (Location‑based)*, *Scope 2 (Market‑based)*, *Scope 3* (case‑insensitive).  
   - If a row is split into sub‑categories (e.g., “Scope 1 – Direct”, “Scope 1 – Stationary combustion”), look for a row that aggregates them (often labelled *Total*, *All activities*, *Combined*, etc.). Use that aggregated value as the **total** for the scope.  
   - If no aggregated row exists, treat each sub‑category as a separate entry for that scope and give it a descriptive `label` (use the exact wording from the row).

2. **Years**  
   - Use the column headings that contain a 4‑digit year.  
   - Do **not** output a year that has no emission number for the given scope.  
   - The evaluator will normalise the year strings; just output the year exactly as it appears.

3. **Values**  
   - Copy the numeric value **exactly as shown** (do not round, convert units, or add commas).  
   - Preserve the decimal point if present; integers are acceptable as numbers.  

4. **Units**  
   - Take the unit that appears in the table header (e.g., `tonnes CO₂e`, `ktCO₂eq`, `t CO2‑e`, `t CO2e`).  
   - Output the unit **exactly** as written, including spaces, superscripts, or hyphens.  

5. **Labels**  
   - If you are outputting the aggregated total for a scope, set `"label"` to `"total"`.  
   - For sub‑category values, set `"label"` to the exact row heading (trim surrounding whitespace).  
   - If no meaningful label exists, use an empty string `""`.

6. **When ambiguous or no total can be identified**  
   - If you cannot find any numeric value for a particular scope‑year pair, **omit** that year from that scope (the evaluator will treat it as “no value”).  
   - Do not fabricate or infer values.

### Output format
Return **only** the following JSON object (replace placeholders with the extracted data):

{
  "report_name": "<filename without .pdf>",
  "report_title": "<full report title>",
  "emissions": {
    "scope_1": {
      "<year>": [ {"value": <number>, "unit": "<string>", "label": "<string>"} , … ]
    },
    "scope_2_location_based": {
      "<year>": [ {"value": <number>, "unit": "<string>", "label": "<string>"} , … ]
    },
    "scope_2_market_based": {
      "<year>": [ {"value": <number>, "unit": "<string>", "label": "<string>"} , … ]
    },
    "scope_3": {
      "<year>": [ {"value": <number>, "unit": "<string>", "label": "<string>"} , … ]
    }
  }
}

*All keys and array structures must be present even if a particular scope has no years (use an empty object `{}` for that scope).  
*Numbers must be raw JSON numbers, not strings.  
*Unit and label must be strings (empty string `""` if unknown).  

Do not include any text before or after the JSON.
