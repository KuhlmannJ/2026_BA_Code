### What must be extracted
- **Scope 1** – direct emissions  
- **Scope 2 – Location‑based**  
- **Scope 2 – Market‑based**  
- **Scope 3** – all other indirect emissions  

For **every year** that appears in the report, locate the row/column that is explicitly labelled as the *total* for the given scope. Acceptable total‑type labels include (but are not limited to):
`Total`, `Grand total`, `Scope X total`, `All emissions`, `Total Scope X`, `Scope X – Total`, etc.  
If a total for a particular year cannot be found with certainty, **do not output that year at all**.

### How to locate the values
1. **Search the whole document** – tables, figures, sidebars, narrative text.  
2. **Identify the table orientation**:  
   * If years appear as **row headers**, the total value sits in the same row as the year.  
   * If years appear as **column headers**, the total value sits in the same column as the year.  
3. **Match the total label** exactly (case‑insensitive) and pair it with the year label.  
4. **Take the number that is directly adjacent to that label** (usually in the same cell).  
5. **Remove any thousands separators** (commas, spaces) but keep decimal points.  
6. **Capture the unit string that appears immediately after the number** (e.g., `ktCO₂eq`, `tonnes CO2e`, `t CO2e`). Do **not** convert or rename units.  
7. **Label field**: if the cell contains an additional descriptive phrase (e.g., “Scope 1 total (including biogenic)”), copy the full phrase into `label`. If no extra phrase is present, set `label` to `""`.

### What to omit
- Any year for which a total cannot be identified with certainty.  
- Numbers that are not explicitly labelled as a total (e.g., “Purchased electricity”, “Scope 2 – Purchased electricity”).  
- Duplicate entries for the same year unless the source provides distinct total rows with different labels.
