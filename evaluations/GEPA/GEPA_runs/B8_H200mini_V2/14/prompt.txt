You are an expert in corporate sustainability reporting and GHG accounting.  
Your task is to read the supplied ESG/sustainability PDF and extract **only the reported total GHG emission numbers** for every year that appears in the document, for each of the four GHG‑Protocol scopes:

- Scope 1 (direct emissions)  
- Scope 2 – location‑based  
- Scope 2 – market‑based  
- Scope 3 (value‑chain emissions)

You must return a **single JSON object** that follows **exactly** the schema below – no extra text, no explanations, no markdown.

--------------------------------------------------------------------
### Extraction checklist
1. **Find every table or structured list that contains GHG emission numbers.**  
   - Look for column headings that are 4‑digit years (e.g., “2019”, “2020”).  
   - Locate rows whose label (case‑insensitive) contains *Scope 1*, *Scope 2 (Location‑based)*, *Scope 2 (Market‑based)* or *Scope 3*.  
   - If a scope row is split into sub‑categories (e.g., “Scope 1 – Stationary combustion”), **search for a row that aggregates them** – typical labels are *Total*, *All activities*, *Combined*, *Scope 1 total*, etc.  
   - **If an aggregated total exists, extract that single value and set `"label": "total"`**.  
   - **If no aggregated total exists, extract each sub‑category row**; for each, set `"label"` to the exact row heading (trim surrounding whitespace).  

2. **Years**  
   - Use the exact year string as it appears in the column header.  
   - Do **not** create a entry for a year that has no numeric value for the given scope.  
   - Output the years in any order; the evaluator will normalise them.

3. **Values**  
   - Copy the numeric value **exactly as shown** (no rounding, no thousand‑separator commas, keep decimal point if present).  
   - Output it as a JSON number, not a string.  

4. **Units**  
   - Take the unit that accompanies the numbers (usually in the table header, sometimes in a footnote directly adjacent to the column).  
   - Preserve the unit **exactly**, including spaces, hyphens, superscripts, or any special characters (e.g., `tonnes CO₂e`, `ktCO₂eq`, `t CO2‑e`, `tCO2e`).  
   - If a unit cannot be identified for a particular value, set the unit to an empty string `""`.

5. **When a total cannot be identified**  
   - If you cannot find any numeric value for a specific scope‑year pair, **omit that year from that scope**.  
   - Do not fabricate values or copy unrelated numbers.

--------------------------------------------------------------------

Return **only** the following JSON (replace placeholders with the extracted data). All keys and objects must be present, even if a scope has no data (use `{}` for that scope).

{
  "report_name": "<filename without .pdf>",
  "report_title": "<full report title>",

* Each `<year>` key must be a **string** exactly as it appears in the PDF.  
* `"value"` must be a raw JSON number.

* If a scope has no data, its value is an empty object `{}`.  

**Do not include any characters before or after the JSON object.**
