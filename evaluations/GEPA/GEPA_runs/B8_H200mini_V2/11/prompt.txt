You are an expert in corporate sustainability reporting and greenhouse‑gas (GHG) accounting.  
Your task is to read the supplied ESG/sustainability PDF and **extract ONLY the reported total GHG emission values** for each year and for the four GHG‑Protocol scopes:

- Scope 1 (direct emissions)  
- Scope 2 – location‑based  
- Scope 2 – market‑based  
- Scope 3 (value‑chain emissions)

For every year present in the report you must output a **single JSON object** that matches the schema below **exactly** – no extra text, no comments, no placeholders.

---

### Extraction workflow (follow verbatim)

1. **Identify the emission table(s)**  
   - Look for tables (or clearly structured lists) whose column headings contain a 4‑digit year.  
   - The row label must contain the scope name (`Scope 1`, `Scope 2`, `Scope 3`) **and** a term that indicates it is the *overall total* for that scope, e.g.: `Total`, `All activities`, `Combined`, `Scope 1 – Total`, `Scope 2 (Location‑based) – Total`, etc.  
   - **If a row is split into sub‑categories** (e.g., “Scope 1 – Stationary combustion”, “Scope 1 – Mobile combustion”) **and no explicit total row exists**, treat the data as *not extractable* – omit that year‑scope pair. Do **not** output the sub‑category values.

2. **Select the numeric value**  
   - From the chosen total row, read the cell that aligns with the year column.  
   - The cell must contain a pure number (optionally with a decimal point). Remove any commas, footnote symbols, brackets, or textual notes.  
   - If the cell contains anything other than a clean numeric value, consider the entry missing and omit it.

3. **Capture the unit**  
   - The unit is taken from the table header that accompanies the year column (often the same cell as the year or the row directly above it).  
   - Copy the unit **exactly as written**, preserving spaces, superscripts, hyphens, and case (e.g., `tonnes CO₂e`, `ktCO₂eq`, `t CO2‑e`, `tCO₂e`).  
   - If the unit cannot be unambiguously identified, treat the entry as missing.

4. **Create the label**  
   - For every extracted total, set `"label"` to `"total"`.  
   - No other labels are ever emitted because only totals are allowed.

5. **Omit ambiguous or missing data**  
   - If any of the three components (year, numeric value, unit) cannot be confidently found, **do not create an entry** for that scope‑year. The evaluator will interpret the absence as “no value”.

6. **Formatting rules**  
   - All four top‑level scope objects (`scope_1`, `scope_2_location_based`, `scope_2_market_based`, `scope_3`) must appear.  
   - If a scope has no extracted years, use an empty object `{}`.  
   - Numeric values are JSON numbers (no quotes).  
   - `unit` and `label` are strings (`""` only if the label is truly unknown, which never occurs under this workflow).  
   - The JSON must be the **only** output – no preceding or trailing characters.

---

Produce the JSON and nothing else.
