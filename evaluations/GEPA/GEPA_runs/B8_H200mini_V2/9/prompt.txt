The output must be a **single valid JSON object** that follows exactly the schema shown at the end of this prompt. No extra text, explanations, or markdown are allowed.

### General extraction strategy (the model must follow this internally)

1. **Find every table (or clearly‑structured list) that contains GHG numbers.**  
   - The table’s rows (or list items) will contain the words **Scope 1**, **Scope 2 (Location‑based)**, **Scope 2 (Market‑based)**, **Scope 3** (case‑insensitive).  
   - The table’s column headings contain the years (four‑digit numbers).  

2. **Determine the unit for each column.**  
   - The unit is taken from the table header that accompanies the numeric values (e.g., `tonnes CO₂e`, `ktCO₂eq`, `t CO2‑e`, `t CO2e`).  
   - Keep the unit **exactly** as it appears, including spaces, hyphens, superscripts, etc.  

3. **Identify the row that represents the **total** for a scope.**  
   - If a row is labelled *Total*, *All activities*, *Combined*, or any synonym that aggregates the sub‑categories, treat that row as the total for the scope and set `"label"` to `"total"`.  
   - If such an aggregated row is **missing**, treat **each sub‑category row** as a separate entry for that scope. Use the exact row heading (trimmed) as the `"label"` value.  

4. **Extract values year‑by‑year.**  
   - For every year column that has a numeric entry for the selected row (total or sub‑category), copy the number **exactly as printed** (do not round, add commas, or change the decimal format).  
   - If a cell is empty or contains a non‑numeric placeholder (e.g., “–”, “n/a”), **skip it** – the evaluator will treat the missing year as “no value”.  

5. **Populate the JSON structure.**  
   - Under each `scope_*` key, create a property for each year that has at least one numeric value. The year must be a **string** exactly as it appears in the table header.  
   - The value for a year is an **array** of objects, one object per extracted entry (total or sub‑category).  
   - Each object must contain three fields:  
     ```json
     {"value": <number>, "unit": "<unit>", "label": "<label>"}
     ```  
   - `"value"` is a JSON number.  
   - `"unit"` is the unit string identified in step 2.  
   - `"label"` is `"total"` for aggregated rows or the exact sub‑category heading; if a label cannot be determined, use `""`.  

6. **When a scope‑year pair has no identifiable total or sub‑category value, simply omit that year from that scope.**  
   - Do **not** insert `null`, `nan`, or placeholder values.  

7. **Do not fabricate, infer, or convert any numbers or units.**  
   - Copy everything verbatim from the source table.
